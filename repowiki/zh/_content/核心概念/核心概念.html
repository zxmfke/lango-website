简介 #

LangGraphGo 是一个基于图结构的工作流引擎，它将应用程序逻辑建模为有向无环图（DAG），其中节点表示处理步骤，边表示数据流向。这种设计提供了高度的灵活性、可组合性和可观测性，特别适用于复杂的 AI 应用程序和数据处理管道。

图结构工作流概述 #

基本架构 #

LangGraphGo 的核心是基于图的执行模型，它将应用程序分解为独立的处理单元（节点），并通过连接这些单元来定义执行流程。

graph TD
A[输入状态] --> B[节点1<br/>数据处理]
B --> C[节点2<br/>业务逻辑]
C --> D[节点3<br/>结果生成]
D --> E[END<br/>结束]
F[状态Schema] --> B
F --> C
F --> D

执行模型 #

图结构工作流采用并行执行模型，在每个步骤中可以同时执行多个节点，然后通过状态合并机制统一处理结果。

G->>N2 : 并行执行
N1-->>G : 返回结果1
N2-->>G : 返回结果2
G->>S : 合并状态
S-->>G : 更新后状态
G->>G : 决定下一跳

节点与边 #

节点（Node） #

节点是图中的基本执行单元，每个节点都有唯一的名称和关联的处理函数。

class Node {

+Function func
+Execute(ctx, state) result
}
class StateRunnable {

  • [state_graph.go](https://github.com/smallnest/langgraphgo/blob/main/graph/state_graph.go#L99-L102)

节点特性 #

  1. 唯一标识: 每个节点必须有唯一的名称
  2. 函数绑定: 关联具体的处理逻辑
  3. 状态感知: 接收当前状态作为输入
  4. 并行执行: 支持多节点并发执行

边（Edge） #

边定义了节点之间的连接关系，控制数据流向和执行顺序。

graph LR
A[开始节点] --> B[处理节点]
B --> C[结束节点]
subgraph "静态边"
A -.->|直接连接| B
B -.->|直接连接| C
end

  • [state_graph.go](https://github.com/smallnest/langgraphgo/blob/main/graph/state_graph.go#L67-L71)

边类型 #

  1. 静态边: 预定义的固定连接关系
  2. 条件边: 基于状态动态决定的连接
  3. 扇出边: 从单个节点到多个目标节点的连接

条件边（Conditional Edge） #

条件边允许根据运行时状态动态选择下一个执行路径。

A[开始] --> B{条件判断}
B --> |条件1| C[路径1]
B --> |条件2| D[路径2]
B --> |默认| E[路径3]
C --> F[结束]
D --> F
E --> F

LangGraphGo 的核心概念构成了一个强大而灵活的工作流引擎，其主要特点包括：

  2. 类型安全: 强类型的状态管理和事件处理
  3. 并发友好: 内置并行执行和状态同步机制
  4. 可观测性: 完整的监听器和流式输出系统

核心价值 #

  1. 容错能力: 检查点系统提供可靠的错误恢复
  2. 可调试性: 丰富的事件系统支持深度调试
  3. 可扩展性: 插件式的监听器和状态管理
  4. 性能优化: 智能的重试和背压处理
